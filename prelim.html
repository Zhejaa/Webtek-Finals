A global system of interconnected computer networks that use the standard Internet Protocol Suite (TCP/IP) to serve billions of users world wide.

It is a network of networks that consists of millions of private, public, academic, business, and government networks, of local to global scope, that are linked by a broad array of electronic, wireless and optical networking technologies.

  • HTTP typically runs on top of TCP/IP, using TCP port 80 by default (TCP port 443 for HTTPS).
  • HTTP resources are identified using URIs (specifically HTTP URLs).
  • HTTP is based on client-server architecture.
    • Clients (AKA User Agents (UA)).
      • Web browsers
      • Web crawlers
      • Email clients
      • Other end user tools and applications
    • Servers
      • Origin servers
      • Proxy servers
      • Gateways
      • Tunnels
  • HTTP uses a request-response standard protocol.
    • Client sends an HTTP request to the server.
    • Server processes the request and replies with an HTTP response message.
  • HTTP is a stateless communications protocol.
    • Servers DO NOT keep information about clients between requests.
    • Web applications affect session tracking using mechanisms such as cookies (a very small text file) on URL-encoded session information to keep track of related client requests.
  • HTTP provides support for other functionalities such as cache-control (cache: Local storage/copy of a resource that is fetched from a server), content media type (MIME) specification, language and character set specification, content/transfer coding, client-server protocol negotiations, persistent connections, request pipelining etc.
